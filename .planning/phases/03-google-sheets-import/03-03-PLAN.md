**Server component: `src/app/(dashboard)/admin/settings/page.tsx`**

1. Import `createClient` from `@/lib/supabase/server`
2. Fetch sheets_config: `supabase.from('sheets_config').select('*').limit(1).single()`
3. If no config exists, pass `null` to the form component (the "not configured" state)
4. Render page with heading "Google Sheets Import" and the SettingsForm component
5. Pass config data as prop: `<SettingsForm config={config} />`

```tsx
import { createClient } from '@/lib/supabase/server'
import { SettingsForm } from './settings-form'

export default async function SettingsPage() {
  const supabase = await createClient()
  const { data: config } = await supabase
    .from('sheets_config')
    .select('*')
    .limit(1)
    .single()

  return (
    <div className="space-y-6">
      <h1 className="text-2xl font-bold">Google Sheets Import</h1>
      <SettingsForm config={config} />
    </div>
  )
}
```

**Client component: `src/app/(dashboard)/admin/settings/settings-form.tsx`**

1. `'use client'` directive
2. Import `useActionState` from `react` (NOT from `react-dom` — Next.js 16 / React 19 exports it from `react`)
3. Import `saveSettingsAction` and `syncFromSheetAction` from `./actions`

**Props interface:**
```ts
interface SheetsConfig {
  id: string
  sheet_url: string
  tab_name: string
  last_synced_at: string | null
  rows_imported: number | null
  last_error: string | null
}

interface SettingsFormProps {
  config: SheetsConfig | null
}
```

**Two separate forms, each with its own useActionState:**

Form 1 — Save Settings:
- `const [saveState, saveAction, savePending] = useActionState(saveSettingsAction, null)`
- Input: Sheet URL (type="url", name="sheet_url", required, defaultValue from config)
- Input: Tab Name (type="text", name="tab_name", defaultValue from config or "Sheet1")
- Submit button: "Save Settings" (disabled while savePending)
- Show success message when `saveState?.success && !saveState?.error`
- Show error message when `saveState?.error`

Form 2 — Sync (only show when config exists):
- `const [syncState, syncAction, syncPending] = useActionState(syncFromSheetAction, null)`
- No visible inputs needed (action reads config from DB)
- Submit button: "Sync Now" (disabled while syncPending)
- Show loading text "Syncing..." while syncPending
- Show error if `syncState?.error`
- Show success if `syncState?.success`

**Status display (below forms, only when config exists):**
- Last synced: format `config.last_synced_at` as readable date, or "Never" if null
- Rows imported: `config.rows_imported ?? 0`
- Last error: show in red text if `config.last_error` exists, otherwise "None"

**UI components to use:**
- `Button` from `@/components/ui/button`
- `Input` from `@/components/ui/input` (if installed) — otherwise use plain `<input>` with Tailwind classes matching existing forms
- `Label` from `@/components/ui/label` (if installed) — otherwise use `<label>` with Tailwind

Check what shadcn components are available in `src/components/ui/` before using them. Use plain HTML + Tailwind for any that don't exist.

**Layout:** Use a card-like container with `border rounded-lg p-6` matching the style of other admin pages. Separate the save form and sync section visually with a horizontal rule or spacing.

**Not-configured state:** When config is null, show the save form with empty fields and a note: "Configure your Google Sheet to start importing clients."
