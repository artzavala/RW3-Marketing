<task type="auto">
  <name>Task 2: Sync from sheet server action</name>
  <files>src/app/(dashboard)/admin/settings/actions.ts</files>
  <action>
Add `syncFromSheetAction(prevState: ActionState, formData: FormData): Promise&lt;ActionState&gt;` to the same file.

Implementation steps:

1. Check admin role. Return `{ error: 'Unauthorized' }` if not admin.

2. Load sheets_config: `adminSupabase.from('sheets_config').select('*').limit(1).single()`. If no config exists, return `{ error: 'No sheet configured. Save settings first.' }`.

3. Extract spreadsheetId from sheet_url using `extractSpreadsheetId()`. If null, return error.

4. Call `readSheetRows(spreadsheetId, config.tab_name)` to get all rows. If empty, return `{ error: 'Sheet is empty or tab not found' }`.

5. Parse rows — row 0 is headers (skip it). For each subsequent row:
   - `sheets_row_id = row[0]?.trim()` — skip row if empty/undefined (no stable key)
   - `name = row[1]?.trim()` — skip row if empty (name is required)
   - `website = row[2]?.trim() || null`
   - `repEmail = row[3]?.trim() || null`

6. Resolve rep emails to profile IDs. Build a cache Map&lt;string, string | null&gt; to avoid repeated lookups:
   ```ts
   const repCache = new Map<string, string | null>()

   async function resolveRep(email: string | null): Promise<string | null> {
     if (!email) return null
     if (repCache.has(email)) return repCache.get(email)!
     const { data: { user } } = await adminSupabase.auth.admin.getUserByEmail(email)
     const repId = user?.id ?? null
     repCache.set(email, repId)
     return repId
   }
   ```

7. Process rows individually (not batch upsert) for partial-failure tolerance. Collect errors:
   ```ts
   const errors: string[] = []
   let imported = 0

   for (const parsed of parsedRows) {
     const assigned_rep = await resolveRep(parsed.repEmail)
     if (parsed.repEmail && !assigned_rep) {
       errors.push(`Row "${parsed.sheets_row_id}": rep email "${parsed.repEmail}" not found`)
     }

     const { error } = await adminSupabase
       .from('clients')
       .upsert(
         {
           sheets_row_id: parsed.sheets_row_id,
           name: parsed.name,
           website: parsed.website,
           assigned_rep,
         },
         { onConflict: 'sheets_row_id' }
       )

     if (error) {
       errors.push(`Row "${parsed.sheets_row_id}": ${error.message}`)
     } else {
       imported++
     }
   }
   ```

8. Update sheets_config with sync results:
   ```ts
   await adminSupabase
     .from('sheets_config')
     .update({
       last_synced_at: new Date().toISOString(),
       rows_imported: imported,
       last_error: errors.length > 0 ? errors.join('; ') : null,
     })
     .eq('id', config.id)
   ```

9. Revalidate `/admin/settings` and `/admin/clients`.

10. Return: if errors exist, `{ error: errors.join('; '), success: true }` (partial success). If no errors, `{ error: '', success: true }`.

IMPORTANT: Use `createAdminClient()` for ALL Supabase calls in this action. The service role key bypasses RLS, which is necessary because:
- sheets_config has admin-only RLS (but we need service-role for auth.admin.getUserByEmail)
- clients upsert needs to set assigned_rep without session-based constraints
- auth.admin endpoints require service role

Wrap the entire sheets read in try/catch to handle API errors (invalid tab name returns 400, auth failures, etc.). On catch, update sheets_config.last_error with the error message and return `{ error: message }`.
