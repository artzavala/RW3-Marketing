# Plan 02-01: Database migration and UI component foundation

## Objective

Create the three database tables (clients, service_packages, client_services) with RLS policies, install required shadcn components, and update the sidebar navigation. This is the foundation that all subsequent Phase 2 plans depend on.

## Context

- Existing migration pattern: `supabase/migrations/0001_create_profiles.sql` -- follow same naming convention
- profiles table has `id UUID` (FK to auth.users), `role TEXT` with CHECK constraint ('admin'|'rep')
- RLS pattern from profiles: `auth.uid()` for user scoping, subquery on `profiles.role` for role checks
- Sidebar is a Client Component at `src/components/app-sidebar.tsx` receiving `role` prop from dashboard layout
- shadcn style is `base-nova` (confirmed in components.json)
- Already installed shadcn components: button, input, sidebar, separator, avatar, dropdown-menu, sheet, skeleton, tooltip

## Tasks

<task id="1" title="Create database migration for clients, service_packages, client_services">

Create file `supabase/migrations/0002_create_clients_services.sql` with the following:

**Tables:**

1. `public.clients` table:
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `name TEXT NOT NULL`
   - `website TEXT`
   - `contact_name TEXT`
   - `contact_email TEXT`
   - `assigned_rep UUID REFERENCES public.profiles(id) ON DELETE SET NULL`
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()`
   - `updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()`

2. `public.service_packages` table:
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `name TEXT NOT NULL`
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()`

3. `public.client_services` junction table:
   - `client_id UUID NOT NULL REFERENCES public.clients(id) ON DELETE CASCADE`
   - `service_package_id UUID NOT NULL REFERENCES public.service_packages(id) ON DELETE CASCADE`
   - `assigned_at TIMESTAMPTZ NOT NULL DEFAULT NOW()`
   - `PRIMARY KEY (client_id, service_package_id)`

**RLS policies (enable RLS on all three tables):**

clients:
- "Admins read all clients" FOR SELECT -- USING subquery: profiles WHERE id = auth.uid() AND role = 'admin'
- "Reps read assigned clients" FOR SELECT -- USING assigned_rep = auth.uid()
- "Admins insert clients" FOR INSERT -- WITH CHECK same admin subquery
- "Admins update clients" FOR UPDATE -- USING same admin subquery
- "Admins delete clients" FOR DELETE -- USING same admin subquery

service_packages:
- "Authenticated users read packages" FOR SELECT -- USING auth.uid() IS NOT NULL
- "Admins write packages" FOR INSERT -- WITH CHECK admin subquery
- "Admins update packages" FOR UPDATE -- USING admin subquery
- "Admins delete packages" FOR DELETE -- USING admin subquery

client_services:
- "Admins read all client_services" FOR SELECT -- USING admin subquery
- "Reps read their client_services" FOR SELECT -- USING EXISTS subquery joining clients WHERE clients.id = client_services.client_id AND clients.assigned_rep = auth.uid()
- "Admins manage client_services" FOR INSERT -- WITH CHECK admin subquery
- "Admins delete client_services" FOR DELETE -- USING admin subquery

**updated_at trigger:**

Add a `BEFORE UPDATE` trigger on `clients` that sets `updated_at = NOW()`. Use a simple PL/pgSQL function (not moddatetime extension, to avoid extension dependency):

```sql
CREATE OR REPLACE FUNCTION public.update_updated_at()
RETURNS TRIGGER
LANGUAGE plpgsql
AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$;

CREATE TRIGGER set_clients_updated_at
  BEFORE UPDATE ON public.clients
  FOR EACH ROW EXECUTE FUNCTION public.update_updated_at();
```

Use the exact SQL from the research doc for RLS policies. Follow the same style as `0001_create_profiles.sql` (comments before each section).
