# Plan 02-03: Client list page and CRUD actions (admin)

## Objective

Build the admin client list page and create/delete actions. Admin sees all clients in a table with rep names and service counts. Admin can add new clients and delete existing ones. Edit and detail view come in Plan 02-04.

Purpose: Satisfies CLI-01 (admin add client), CLI-03 (admin remove client), CLI-04 (admin view all clients).

## Context

- Route: `src/app/(dashboard)/admin/clients/` -- already protected by proxy.ts
- Use `createClient()` from `@/lib/supabase/server` for queries (RLS gives admin full SELECT)
- Use `createAdminClient()` from `@/lib/supabase/admin` ONLY for the rep dropdown query (profiles RLS blocks cross-user reads)
- PostgREST FK join syntax for rep name: `profiles!clients_assigned_rep_fkey(id, name)`
- PostgREST aggregation for service count: `client_services(count)`
- `redirect()` outside try/catch in actions
- Server Actions return `{ error: string } | null` type for form error display
- `searchParams` in Next.js 16 page components is a Promise -- must be awaited

## Tasks

<task id="1" title="Create Server Actions for client CRUD">

Create `src/app/(dashboard)/admin/clients/actions.ts` with `'use server'` directive.

**Type at top:**
```typescript
type ActionState = { error: string } | null
```

**Three actions:**

1. `createClientAction(_prev: ActionState, formData: FormData): Promise<ActionState>`
   - `checkRole('admin')` guard -> return `{ error: 'Unauthorized' }` if false
   - Extract fields: `name` (required, trim), `website` (optional), `contact_name` (optional), `contact_email` (optional), `assigned_rep` (optional UUID string)
   - Validate: name must be non-empty after trim
   - Insert into `clients` table. For `assigned_rep`, pass `null` if empty string
   - If error, return `{ error: error.message }`
   - `revalidatePath('/admin/clients')`
   - `redirect('/admin/clients')` -- outside try/catch

2. `updateClientAction(_prev: ActionState, formData: FormData): Promise<ActionState>`
   - `checkRole('admin')` guard
   - Extract `id` + same fields as create
   - Update `clients` row by id
   - `revalidatePath('/admin/clients')`
   - `revalidatePath('/admin/clients/' + id)` -- also revalidate detail page
   - `redirect('/admin/clients')` -- outside try/catch

3. `deleteClientAction(id: string): Promise<ActionState>`
   - `checkRole('admin')` guard
   - Delete from `clients` where id matches
   - `revalidatePath('/admin/clients')`
   - Return null on success, `{ error: message }` on failure
   - No redirect (stays on list, page revalidates)

**Imports:**
- `createClient` from `@/lib/supabase/server`
- `checkRole` from `@/lib/roles`
- `revalidatePath` from `next/cache`
- `redirect` from `next/navigation`

</task>

<task id="2" title="Build admin client list page and new client form" depends_on="1">

**File 1: `src/app/(dashboard)/admin/clients/page.tsx`** (Server Component)

Query clients with joined rep name and service count:
```typescript
const supabase = await createClient()
const { data: clients } = await supabase
  .from('clients')
  .select(`
    id,
    name,
    website,
    assigned_rep,
    profiles!clients_assigned_rep_fkey ( id, name ),
    client_services ( count )
  `)
  .order('name')
```

Render:
- Page heading "Clients" with a link button "Add Client" pointing to `/admin/clients/new`
- shadcn `Table` with columns: Name (linked to `/admin/clients/[id]`), Website, Assigned Rep, Services, Actions
- Name column: render as `<Link href={'/admin/clients/' + client.id}>` so it navigates to detail (detail page built in 02-04)
- Assigned Rep column: show `client.profiles?.name ?? '—'`
- Services column: show count from `client.client_services?.[0]?.count ?? 0`
- Actions column: Delete button (confirmation via `window.confirm`, calls `deleteClientAction`)

The Delete button needs to be a Client Component (it calls a Server Action on click). Extract a small `DeleteClientButton` Client Component (can live in a separate file like `components.tsx` in the same directory, or inline -- Claude's choice).

**Empty state:** If no clients, show "No clients yet" with the Add Client link.

**File 2: `src/app/(dashboard)/admin/clients/new/page.tsx`** (Server Component wrapping Client form)

This page fetches the rep list and renders a create form.

Server Component part:
```typescript
import { createAdminClient } from '@/lib/supabase/admin'

export default async function NewClientPage() {
  const admin = createAdminClient()
  const { data: reps } = await admin
    .from('profiles')
    .select('id, name, email')
    .eq('role', 'rep')
    .order('name')

  return <ClientForm reps={reps ?? []} />
}
```

Client Component `ClientForm`:
- Uses `useActionState(createClientAction, null)` for form submission
- Fields: Name (required Input), Website (Input), Contact Name (Input), Contact Email (Input type="email"), Assigned Rep (shadcn Select with options from reps prop + an empty "None" option)
- Display `state.error` as red text above submit button
- Submit button shows "Creating..." when isPending
- Cancel link back to `/admin/clients`

**Rep Select component:** Use shadcn `Select` with `SelectTrigger`, `SelectValue`, `SelectContent`, `SelectItem`. The hidden input `name="assigned_rep"` must carry the selected value for FormData extraction. shadcn Select does not natively submit form data -- use a pattern like a hidden input updated via `onValueChange`, OR wrap the Select value in the form action manually. Simplest: use a controlled state for the rep value, render a `<input type="hidden" name="assigned_rep" value={selectedRep} />`.
