# Plan 02-02: Service packages admin CRUD

## Objective

Build the service packages management page where admins can create, edit, and delete service packages. This is a self-contained vertical slice: Server Actions for mutations + a Server Component page with Client Component dialogs for add/edit.

Purpose: Satisfies SVC-01, SVC-02, SVC-03. Service packages must exist before they can be assigned to clients in Plan 02-04.

## Context

- Route lives at `src/app/(dashboard)/admin/services/` -- protected by proxy.ts admin guard at routing layer
- Server Actions still need `checkRole('admin')` defense-in-depth
- Use `createClient()` from `@/lib/supabase/server` for all queries (RLS allows authenticated read on service_packages)
- `redirect()` must be called OUTSIDE try/catch blocks (Next.js 16 throws NEXT_REDIRECT)
- Server Actions should return `{ error: string } | null` for form error display, not throw on validation failures
- Use `useActionState` (React 19) for form state management in Client Components
- Use shadcn `Dialog`, `Table`, `Input`, `Label`, `Button` components
- Use `sonner` toast for success feedback: `import { toast } from 'sonner'`
- `revalidatePath('/admin/services')` after each mutation

## Tasks

<task id="1" title="Create Server Actions for service package CRUD">

Create `src/app/(dashboard)/admin/services/actions.ts` with `'use server'` directive.

**Three actions:**

1. `createPackageAction(_prev: ActionState, formData: FormData): Promise<ActionState>`
   - `checkRole('admin')` guard, return `{ error: 'Unauthorized' }` if not admin
   - Extract `name` from formData, trim it, validate non-empty (return `{ error: 'Package name is required' }`)
   - Insert into `service_packages` table via server supabase client
   - If DB error, return `{ error: error.message }`
   - `revalidatePath('/admin/services')`
   - `redirect('/admin/services')` -- OUTSIDE try/catch

2. `updatePackageAction(_prev: ActionState, formData: FormData): Promise<ActionState>`
   - `checkRole('admin')` guard
   - Extract `id` and `name` from formData
   - Validate name non-empty
   - Update `service_packages` where `id` matches
   - `revalidatePath('/admin/services')`
   - `redirect('/admin/services')`

3. `deletePackageAction(id: string): Promise<ActionState>`
   - `checkRole('admin')` guard
   - Delete from `service_packages` where `id` matches
   - `revalidatePath('/admin/services')`
   - Return null on success, `{ error: message }` on failure
   - Do NOT redirect (dialog closes, page revalidates)

**Type definition at top of file:**
```typescript
type ActionState = { error: string } | null
```

**Imports needed:**
- `createClient` from `@/lib/supabase/server`
- `checkRole` from `@/lib/roles`
- `revalidatePath` from `next/cache`
- `redirect` from `next/navigation`

</task>

<task id="2" title="Build service packages list page with add/edit/delete dialogs" depends_on="1">

Create `src/app/(dashboard)/admin/services/page.tsx` as a Server Component.

**Server Component (page.tsx) responsibilities:**
- Query `service_packages` ordered by `name` via server supabase client
- Render page heading "Service Packages" with an "Add Package" button (opens dialog)
- Render a `Table` of packages with columns: Name, Created, Actions
- Pass data to Client Components for interactive parts

**Client Components (can be in same file or a separate `components.tsx` in the same directory -- Claude's choice):**

1. `AddPackageDialog` -- Dialog with form containing:
   - `Input` for package name (required)
   - Submit button using `useActionState(createPackageAction, null)`
   - Display `state.error` if present
   - Use a key prop or open state counter to reset form on reopen (see pitfall 5 in research)

2. `EditPackageDialog` -- Dialog receiving current package `{ id, name }` as props:
   - Pre-filled `Input` with current name
   - Hidden input for `id`
   - Submit via `useActionState(updatePackageAction, null)`

3. `DeletePackageButton` -- Button that calls `deletePackageAction(id)` on click:
   - Show confirmation via `window.confirm()` (simple, sufficient for v1 internal tool)
   - On success, toast "Package deleted"
   - On error, toast error message

**Table layout:**
```
| Name           | Created      | Actions        |
|----------------|-------------|----------------|
| SEO            | Apr 1, 2026 | [Edit] [Delete]|
```

Format `created_at` as a readable date string. Use `Intl.DateTimeFormat` or `toLocaleDateString`.

**Empty state:** If no packages, show a centered message "No service packages yet" with the Add Package button.

</task>
