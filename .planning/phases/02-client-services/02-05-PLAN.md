# Plan 02-05: Rep client list and detail pages (read-only)

## Objective

Build the rep-facing client views. Reps see only their assigned clients (enforced by RLS) and can view client details including assigned service packages. No edit controls are shown.

Purpose: Satisfies CLI-04 (rep views accessible client list), CLI-05 (rep views client detail), SVC-05 (rep sees which packages client uses), AUTH-05 (rep only sees assigned clients).

## Context

- Route: `src/app/(dashboard)/rep/clients/` -- no proxy guard needed (rep is the default role)
- RLS on `clients` table: rep SELECT policy filters `assigned_rep = auth.uid()` -- the query automatically returns only assigned clients
- RLS on `client_services`: rep SELECT policy filters via join to `clients.assigned_rep = auth.uid()`
- Use `createClient()` from `@/lib/supabase/server` -- RLS does all the scoping
- These pages are pure Server Components -- no Client Components needed (no forms, no mutations)
- Can reuse the same Table component patterns from admin pages but without action columns
- Service packages are readable by all authenticated users (RLS policy)

## Tasks

<task id="1" title="Build rep client list page">

Create `src/app/(dashboard)/rep/clients/page.tsx` as a Server Component.

**Query:**
```typescript
const supabase = await createClient()
const { data: clients } = await supabase
  .from('clients')
  .select(`
    id,
    name,
    website,
    client_services ( count )
  `)
  .order('name')
```

RLS automatically filters to `assigned_rep = auth.uid()`. No manual WHERE clause needed.

**Render:**
- Page heading "My Clients"
- shadcn `Table` with columns: Name (linked to `/rep/clients/[id]`), Website, Services (count)
- No Actions column (reps have no write permissions)
- Name as `<Link href={'/rep/clients/' + client.id}>`

**Empty state:** "No clients assigned to you yet." -- centered text, no action button (reps can't create clients).

</task>

<task id="2" title="Build rep client detail page">

Create `src/app/(dashboard)/rep/clients/[id]/page.tsx` as a Server Component.

**Queries:**

1. Client data:
```typescript
const { id } = await params
const supabase = await createClient()
const { data: client } = await supabase
  .from('clients')
  .select('id, name, website, contact_name, contact_email')
  .eq('id', id)
  .single()
```

If client is null (RLS blocks access or doesn't exist), render "Client not found" or call `notFound()`. This handles the case where a rep tries to access a client not assigned to them.

2. Assigned services with package names:
```typescript
const { data: services } = await supabase
  .from('client_services')
  .select(`
    service_package_id,
    service_packages ( name )
  `)
  .eq('client_id', id)
```

**Page layout (read-only, two sections):**

**Section 1: Client Information**
- Display card with: Name, Website (as link if present), Contact Name, Contact Email
- All read-only text, no form inputs
- Back link to `/rep/clients`

**Section 2: Assigned Services**
- Heading "Services"
- List of assigned service package names as `Badge` components (from shadcn)
- If no services assigned, show "No services assigned"

No edit buttons, no forms, no delete actions.

</task>
