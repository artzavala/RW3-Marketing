<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Provision Neon database and configure Clerk</name>
  <action>The user must complete these external service setup steps.</action>
  <instructions>
    Complete these steps in order:

    **A. Provision Neon Postgres (via Vercel Marketplace)**
    1. Go to https://vercel.com/dashboard
    2. Select (or create) the project for this app
    3. Go to Storage tab -> Browse Marketplace -> Neon Serverless Postgres
    4. Create a new database (name: `client-intelligence`)
    5. This auto-injects `DATABASE_URL` and `DATABASE_URL_UNPOOLED` into your Vercel project env vars
    6. Copy both connection strings to your local `.env.local` file, replacing the PLACEHOLDER values

    **B. Configure Clerk Application**
    1. Go to https://dashboard.clerk.com
    2. Create a new application (or use existing) named "Client Intelligence Platform"
    3. Under **User & Authentication -> Email, Phone, Username**: Enable Email address + Password
    4. Under **API Keys**: Copy `Publishable Key` and `Secret Key` to `.env.local`
    5. Under **Sessions -> Customize session token**: Click "Edit" and set the claims to:
       ```json
       {
         "metadata": "{{user.public_metadata}}"
       }
       ```
       This is CRITICAL — without this, role-based routing will not work.
    6. Under **Webhooks -> Add Endpoint**:
       - URL: `https://YOUR-VERCEL-URL.vercel.app/api/webhooks/clerk` (use your deployed URL, or use ngrok for local dev)
       - Events: Select `user.created`, `user.updated`, `user.deleted`
       - Copy the **Signing Secret** to `.env.local` as `CLERK_WEBHOOK_SIGNING_SECRET`

    **C. Verify `.env.local` is complete**
    All these should have real values (not PLACEHOLDER):
    - `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY`
    - `CLERK_SECRET_KEY`
    - `CLERK_WEBHOOK_SIGNING_SECRET`
    - `DATABASE_URL`
    - `DATABASE_URL_UNPOOLED`
  </instructions>
  <resume-signal>Type "done" when all Clerk and Neon setup is complete and .env.local has real values.</resume-signal>
</task>

<task type="auto">
  <name>Task 2: Run database migration to create users table</name>
  <files></files>
  <action>
    After the user confirms external setup is complete, run the database migration:

    1. Generate the migration from the Drizzle schema:
    ```bash
    npm run db:generate
    ```
    This reads `src/db/schema.ts` and generates SQL migration files in `./drizzle/`.

    2. Push the schema to the database:
    ```bash
    npm run db:push
    ```
    This applies the schema directly to the Neon database. Use `db:push` for development (faster than `db:migrate`).

    3. Verify the table exists by opening Drizzle Studio:
    ```bash
    npm run db:studio
    ```
    Or verify via the Neon dashboard SQL editor: `SELECT * FROM users;` should return empty results (no error).

    4. Verify the app starts and connects:
    ```bash
    npm run dev
    ```
    The app should start without database connection errors.
  </action>
  <verify>
    - `npm run db:push` completes without errors
    - Migration files exist in `./drizzle/` directory
    - `npm run dev` starts without errors
  </verify>
